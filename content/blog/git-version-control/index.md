## Что такое Git?

**Git** — это распределённая система контроля версий, созданная Линусом Торвальдсом в 2005 году. Она позволяет отслеживать изменения в файлах, работать над проектом совместно с другими разработчиками и при необходимости откатываться к предыдущим версиям кода.

## Основные концепции

### Репозиторий

Репозиторий (repository) — это хранилище вашего проекта, включающее все файлы и историю их изменений. Репозитории бывают:
- **Локальные** — на вашем компьютере
- **Удалённые** — на сервере (GitHub, GitLab, Bitbucket)

### Коммит

Коммит (commit) — это снимок состояния проекта в определённый момент времени. Каждый коммит содержит:
- Уникальный идентификатор (хеш)
- Сообщение с описанием изменений
- Информацию об авторе
- Ссылку на родительский коммит

### Ветки

Ветка (branch) — это независимая линия разработки. Основная ветка обычно называется `main` или `master`.

## Основные команды Git

### Начало работы

```bash
# Инициализация нового репозитория
git init

# Клонирование существующего репозитория
git clone <url>

# Настройка имени и email
git config --global user.name "Ваше Имя"
git config --global user.email "email@example.com"
```

### Работа с изменениями

```bash
# Проверка статуса
git status

# Добавление файлов в индекс
git add <файл>
git add .  # добавить все файлы

# Создание коммита
git commit -m "Описание изменений"

# Просмотр истории коммитов
git log
git log --oneline  # краткий формат
```

### Работа с ветками

```bash
# Создание новой ветки
git branch <имя-ветки>

# Переключение на ветку
git checkout <имя-ветки>

# Создание и переключение одной командой
git checkout -b <имя-ветки>

# Слияние веток
git merge <имя-ветки>
```

### Работа с удалённым репозиторием

```bash
# Добавление удалённого репозитория
git remote add origin <url>

# Отправка изменений
git push origin <ветка>

# Получение изменений
git pull origin <ветка>

# Получение без слияния
git fetch
```

## Рабочий процесс Git

Типичный рабочий процесс выглядит так:

1. **Создание ветки** для новой функциональности
2. **Внесение изменений** в код
3. **Коммит изменений** с описательным сообщением
4. **Пуш** ветки на удалённый репозиторий
5. **Создание Pull Request** для ревью кода
6. **Слияние** в основную ветку после одобрения

## Лучшие практики

- Делайте **небольшие, атомарные коммиты**
- Пишите **понятные сообщения** коммитов
- Используйте **ветки** для новых функций
- Регулярно **синхронизируйтесь** с удалённым репозиторием
- Не коммитьте **секретные данные** (пароли, ключи API)

## Заключение

Git — незаменимый инструмент современного разработчика. Освоение его базовых команд значительно повышает продуктивность и позволяет эффективно работать в команде. Начните с простых операций и постепенно изучайте более продвинутые возможности.
